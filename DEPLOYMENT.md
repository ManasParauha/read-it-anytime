# Production Deployment Guide (Vercel + Supabase + Inngest)

Follow these steps to deploy **Read It Anytime** to production.

---

## Step 1: Database Setup & Migrations on Supabase

When deploying to production, database tables must be migrated to your Supabase project.

1. Retrieve the **Direct Connection String** from your Supabase Project Settings (Settings -> Database -> Connection string -> URI, choose "Direct" or port 5432).
2. Set this value locally as `DIRECT_URL`.
3. Retrieve the **Transaction Connection String** (port 6543, with pooler routing enabled) and set it locally as `DATABASE_URL`.
4. Run the Prisma migration deploy command to apply existing migrations on the remote production Supabase database:
   ```bash
   npx prisma migrate deploy
   ```
5. *(Optional)* Generate the Prisma client locally for runtime type matching:
   ```bash
   npx prisma generate
   ```

## Step 3: Register and Configure Inngest Cloud

Inngest handles the background processing of links and the weekly cron schedule.

1. Create a free account on [Inngest Cloud](https://www.inngest.com/).
2. Connect your production Vercel deployment by registering a new **App**.
3. Provide the full URL path pointing to the handler endpoint:
   ```
   https://your-app-domain.vercel.app/api/inngest
   ```
4. Copy the **Signing Key** (`signkey_...`) generated by Inngest Cloud and add it to Vercel as `INNGEST_SIGNING_KEY` (and `INNGEST_EVENT_KEY` for event delivery).
5. Deploy or redeploy the project on Vercel so the signing key takes effect.
6. Trigger a sync in Inngest Cloud. Once synced:
   - The weekly cron job (`weekly-digest-cron`) is automatically detected by Inngest and schedules itself according to the trigger rule `0 8 * * 1` (Monday at 8:00 AM UTC).
   - You can monitor execution runs, view step details, and trigger test runs directly via the Inngest Cloud dashboard.

---

## Step 4: Verification and Uptime Monitoring

1. Visit your app's public health route to confirm the database and server are healthy:
   ```
   https://your-app-domain.vercel.app/api/health
   ```
2. Integrate this endpoint with uptime monitoring services (like UptimeRobot, Better Stack, or Vercel's native checks) to ensure continuous database connectivity.
